# 每日產業預測快照設計

## 目標

把目前 LINE「產業預測」從固定排序改成每日更新的市場快照。使用者點選產業時，LINE 直接回覆當天已計算好的推薦清單，不在 Webhook 內即時掃描多檔股票。

## 不做的事

- 不全市場逐檔每日回測。Cloud Run 只有 1GB RAM，這會增加 timeout、FinMind 配額與冷啟動風險。
- 不新增重型資料庫或排程框架。沿用 Cloud Scheduler、Flask endpoint、Firestore REST 與現有 `analyze()`。
- 不改 LightGBM 模型核心。先把產業候選股與快照機制補齊。

## 架構

```text
Cloud Scheduler
  -> POST /tasks/refresh-sector-signals
  -> 驗證 ALERT_TASK_TOKEN
  -> 取得 twstock 產業分類
  -> 每個產業挑最多 20 檔候選股
  -> 逐檔執行現有 analyze()
  -> 依 AI 勝率、回測、外資、成交量排序
  -> 儲存每日 sector_signals 快照
  -> LINE 產業預測只讀快照
```

## 候選股範圍

每個產業最多分析 20 檔。候選股來源仍使用 `twstock.codes` 的產業分類，避免引入新依賴。

候選股初版規則：

- 排除明顯不是一般股票的項目；ETF 留在 `ETF專區`。
- 使用產業內前 20 檔作為分析上限，避免排程失控。
- 若後續要更貼近成交活躍度，可在不改 LINE 介面的前提下，把候選挑選改成「先抓近期成交量再挑前 20」。

## 排序分數

每日快照內每檔股票保留：

- 股票代碼與名稱
- 最新收盤價
- 五日上漲機率
- 趨勢文字
- 回測策略累積報酬
- 回測最大回檔
- 外資近 5 日買賣超
- 資料日期

排序使用一個簡單加權分數：

```text
score = AI 勝率
      + 回測報酬加分
      + 外資近 5 日買超加分
      - 最大回檔懲罰
```

這個分數只用來排序，不對使用者宣稱為保證績效。

## 資料儲存

新增一份每日產業快照，格式如下：

```json
{
  "as_of": "2026-06-24",
  "generated_at": "2026-06-24T08:30:00Z",
  "sectors": {
    "半導體": [
      {
        "code": "2330",
        "name": "台積電",
        "price": 1000.0,
        "prob": 63,
        "trend": "多頭",
        "score": 71.2,
        "strat_cum": 12.5,
        "mdd": -8.3,
        "foreign_net_5": 12345.0,
        "as_of": "2026-06-24"
      }
    ]
  }
}
```

儲存位置優先使用 Firestore。若 Firestore 未設定，排程回傳 503，不在記憶體內假裝完成，避免 Cloud Run 重啟後資料消失。

## LINE 行為

- 使用者輸入 `預測` 或點圖文選單後，仍顯示產業 quick reply。
- 使用者選擇 `選產業_<名稱>`：
  - 有今日或最新快照：顯示排序後前 10 檔。
  - 無快照：回覆「產業資料尚未更新，請稍後再試」，並保留原本查個股方式。
- 每張股票卡保留「查詢個股」與「完整分析」入口。

## 錯誤處理

- 單一股票分析失敗：略過該股票，不中斷整個產業。
- 單一產業全部失敗：該產業存空清單與錯誤摘要。
- 排程整體失敗：回傳 500，不推送 LINE 訊息。
- Token 錯誤：回傳 403。

## 測試

新增最小測試：

- 排序分數會讓高勝率、外資買超、回測較好的股票排前面。
- 排程只分析每個產業最多 20 檔。
- LINE 選產業時優先讀快照，不同步跑大量 `analyze()`。
- 沒有快照時回覆可理解的降級訊息。

## 部署設定

Cloud Scheduler 每個交易日收盤後呼叫：

```text
POST /tasks/refresh-sector-signals
Authorization: Bearer ${ALERT_TASK_TOKEN}
```

沿用 `ALERT_TASK_TOKEN`，不新增第二組排程 token。

## 成功標準

- LINE Webhook 不因產業預測而變慢。
- 每日產業推薦可以反映最新資料日期。
- 1GB RAM Cloud Run 不需要新增重型依賴。
- 測試通過後才能 push。
