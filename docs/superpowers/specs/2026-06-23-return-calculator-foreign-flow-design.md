# 投資金額試算與外資買賣超判讀設計

日期：2026-06-23

## 目標

在不增加新套件、不增加資料庫、不改 LINE webhook 核心架構的前提下，讓使用者能在個股完整分析頁看懂兩件事：

1. 如果投入一筆金額，依目前收盤價大約能買幾股，並用現有回測績效換算歷史損益。
2. 外資近期是偏買、偏賣或中性，作為投資標的判斷的籌碼輔助訊號。

LINE 端先維持輕量，Web 測試通過後再補「按鈕優先」的投資試算流程；文字指令只保留為備援。

## 範圍

### Phase A：Web 個股頁

修改 `/stock/<code>`：

- 新增「投資金額試算」區塊。
- 新增「外資買賣超」區塊。
- 保留既有圖表、技術指標、模型解釋、風險提醒與新聞區塊。

### Phase B：LINE 按鈕式簡短試算

股票查詢結果新增一個 `投資試算` 按鈕。使用者點擊後，LINE 回覆預設金額選項：

- `1 萬`
- `5 萬`
- `10 萬`
- `自訂金額`

使用者點擊預設金額後，直接回覆一張簡短 Flex 卡，包含：

- 股票名稱與最新收盤價
- 約可買股數
- AI 策略歷史估算損益
- 買進持有歷史估算損益
- `查看完整分析` CTA

`自訂金額` 才提示使用者輸入 `試算 2330 100000`。這是備援，不是主要路徑。

不為預設金額新增 pending state；只有自訂金額沿用文字指令解析。

## 資料與計算

### 投資金額試算

後端新增一個小型計算函式，使用既有 `analyze()` 結果：

- `shares = floor(amount / latest_close)`
- `deployed_amount = shares * latest_close`
- `strategy_profit = deployed_amount * bt.strat_cum / 100`
- `buy_hold_profit = deployed_amount * bt.bh_cum / 100`
- `strategy_annualized = CAGR(bt.strat_cum, bt.days)`
- `buy_hold_annualized = CAGR(bt.bh_cum, bt.days)`

CAGR 使用交易日估算：

```text
annualized = (1 + total_return) ** (252 / trading_days) - 1
```

若金額無效、股數為 0、回測天數不足或總報酬小於等於 -100%，顯示「無法試算」，不讓整頁失敗。

Web 前端用原生 `<input type="number">` 即時計算不同投入金額，不打 API。預設金額為 `100000`。

### 外資買賣超

延伸既有 FinMind `TaiwanStockInstitutionalInvestorsBuySell` 資料處理：

- 優先從資料中的投資人類別拆出外資淨買賣超。
- 若資料 schema 無法辨識外資類別，退回使用現有三大法人淨買賣超作為籌碼近似值。
- 新增欄位：
  - `ForeignNet`
  - `ForeignNet5`
  - `ForeignNet20`
  - `ForeignFlowStatus`

判讀規則保持簡單：

- 近 5 日與近 20 日皆大於 0：外資偏多
- 近 5 日與近 20 日皆小於 0：外資偏空
- 其他：外資中性
- 無資料：資料不足

這個訊號只作為展示與輔助判讀，不直接修改模型機率。

## UI 設計

### Web

在個股頁圖表下方加入兩個 panel：

1. 「投資金額試算」
   - 金額輸入框
   - 約可買股數
   - AI 策略估算損益
   - 買進持有估算損益
   - 年化報酬率比較
   - 小字註明：歷史回測換算，不代表未來獲利

2. 「外資買賣超」
   - 近 5 日外資淨買賣超
   - 近 20 日外資淨買賣超
   - 判讀標籤：偏多 / 偏空 / 中性 / 資料不足
   - 小字註明：籌碼資料可能延遲或缺漏

### LINE

保留目前「關注、提醒、強勢訊號在 LINE；完整分析在 Web」的分工。

主要流程使用按鈕：

1. 使用者查詢 `2330`。
2. 股票 Flex 卡提供 `投資試算`。
3. 點擊後出現預設金額按鈕。
4. 點擊金額後回覆試算摘要。

文字指令 `試算 2330 100000` 只作為自訂金額備援；完整內容導到 `/stock/2330`。

## 錯誤處理

- 金額不是數字、金額小於等於 0：LINE 回覆預設金額按鈕與格式範例，Web 顯示輸入提示。
- 股票查無資料：沿用既有查詢失敗訊息。
- 外資資料缺漏：顯示「資料不足」，不影響價格、模型與新聞。
- 回測資料不足：顯示「無法試算」，不影響其他區塊。

## 測試

新增或更新既有測試：

- 投資試算函式：
  - 正常金額能算股數、投入金額與兩種損益。
  - 金額不足買 1 股時回傳不可試算。
  - 年化報酬率處理負報酬與不足資料。
- 外資資料處理：
  - 可拆出外資時使用外資。
  - 無法拆出外資時退回三大法人。
  - 缺資料時回中性或資料不足。
- Web：
  - `/stock/2330` 包含投資試算與外資買賣超區塊。
  - 前端 bundle 包含金額試算初始化。
- LINE：
  - 股票卡包含 `投資試算` 按鈕。
  - 投資試算按鈕回覆預設金額選項。
  - 預設金額 postback 回覆 Flex。
  - `試算 2330 100000` 備援指令回覆 Flex。
  - 格式錯誤時回覆使用範例。

## 不做的事

- 不做登入或個人投資紀錄保存。
- 不做「保證獲利」或「建議買進金額」文字。
- 不改模型訓練架構；外資訊號先作為展示，不重新設計模型。
