# Stock Papi 網站功能豐富度與直覺性強化設計

## 目標

讓第一次進入 Stock Papi 網站的使用者，在十秒內完成三件事：知道網站能做什麼、找到想看的股票、理解個股分析的閱讀順序。維持 LINE 負責關注與提醒，Web 負責市場與個股詳細分析。

## 設計策略

沿用現有 Flask、Jinja、Vanilla JS、CSS 與 Lightweight Charts，不加入前端框架或新依賴。保留暖白、薄荷、杏桃、薰衣草與節制液態玻璃風格，主要改善資訊架構與互動提示。

## 首頁

### 真正可用的搜尋入口

- Cloud Run 根網址直接顯示 Dashboard，不再只顯示健康文字。
- Hero 加入原生 GET 搜尋表單，接受台股代碼、名稱及美股代碼。
- 搜尋成功導向 `/stock/<code>`；找不到時回到 Dashboard，保留輸入值並顯示清楚錯誤。
- 提供 `2330`、`台積電`、`AAPL` 等格式範例，不加入即時 autocomplete API。

### 快速理解與定位

- Hero 明確說明「輸入股票後查看五日機率、圖表、回測與新聞情緒」。
- 桌機側欄與頁內快捷列連到市場摘要、產業預測、精選標的與新手說明。
- 市場摘要增加資料日期、情緒狀態與一句白話判讀。
- 產業卡保留領先股，增加排名、資料日期與外資方向，避免只看到一個機率。

## 個股頁

### 閱讀順序

1. 股票名稱、資料日期與五日結論。
2. 預測摘要與一段 Papi 判讀。
3. 價格圖表。
4. 投資試算、技術面、外資與回測。
5. 情緒拆解、新聞、新手解讀與風險。

### 新增互動

- 頁首加入頁內導覽：摘要、圖表、試算、回測、情緒、風險。
- 投資試算加入 1 萬、5 萬、10 萬按鈕，仍保留自訂數字輸入。
- 新增回測摘要卡，顯示策略報酬、買進持有、勝率、最大回撤、交易數與 Brier Score。
- 情緒區顯示目前已有的動能、波動、分歧、有效樣本、發布者數與 metadata 完整度。
- 新聞提供全部、正向、負向、中性篩選；沒有符合項目時顯示空狀態。

## 手機版

- 頁內導覽改成可橫向捲動的黏性快捷列，不壓縮文字。
- 搜尋框與主要按鈕至少 44px 高。
- 摘要、回測與情緒卡在 760px 以下改為單欄。
- 圖表延用既有寬度與 `ResizeObserver` 邊界，不增加橫向捲動。
- 手機底部仍只保留市場首頁與大盤分析，不加入 Web 關注或提醒操作。

## 錯誤與降級

- Dashboard API 失敗時保留搜尋與靜態說明，只隱藏動態資料。
- 搜尋空字串或未知股票不觸發分析。
- JavaScript 停用時搜尋、頁內錨點、`details` 與主要內容仍可使用。
- 情緒資料不足時顯示「資料不足」，不把 0 誤解為中性或沒有風險。

## 不做

- 不新增 Web 關注清單、提醒或登入。
- 不新增股票比較器、全市場篩選器或 SPA 路由。
- 不增加新的行情、新聞或模型 API 呼叫。
- 不改預測機率、模型或排程。

## 測試與成功標準

- `/` 與 `/dashboard` 都回傳完整產品首頁。
- 搜尋台股代碼、名稱與美股代碼正確導向；未知股票安全返回並提示。
- Dashboard 包含真實搜尋、快捷導覽與市場資料日期。
- 個股頁包含頁內導覽、金額按鈕、回測與情緒拆解。
- 新聞篩選不使用 `innerHTML` 寫入外部新聞內容，只切換既有 DOM 顯示狀態。
- 390px 手機寬度沒有水平溢位，圖表與卡片不遮擋。
- 鍵盤焦點、ARIA、reduced motion 與無 JavaScript 基本閱讀能力維持。
